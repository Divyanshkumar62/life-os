PRD UPDATE: The "Job Change" Quest (Mid-Game Evolution) (v2.0)
1. Objective
Implement a high-stakes, multi-day progression event triggered at Level 40. This event overrides the standard daily loop, forces a survival gauntlet, and analyzes the player's historical stats to assign a permanent "Class" (Job) that grants specific multipliers and thematic UI overhauls.

2. The Trigger Condition (The Summons)
Context: The System does not ask if the player is ready. When the threshold is met, the System demands evolution.

Requirements:

The Level Gate: The event strictly triggers the moment current_level reaches 40.

The UI Takeover: * The standard Dashboard is completely locked out.

The screen transitions to a dark, pulsing portal aesthetic.

System Voice Modal: "Requirement met. The Job Change Quest has been generated. Do you accept?" (Note: There is no "No" button. The only options are "Accept" or "Delay for 24h").

State Change: player_identity.promotion_status changes to JOB_CHANGE_IN_PROGRESS. Active projects are temporarily SUSPENDED.

3. The Gauntlet Mechanics (The 3-Day Event)
Context: The Job Change is a 3-day endurance test mirroring Jin-Woo's fight against the endless armor knights and Igris.

Requirements:
The DungeonArchitectService overrides the standard Daily Quests with the following fixed structure:

Day 1: "The Endless Swarm" (Volume Test)

Quests: 6 to 8 Low-Difficulty / Low-Time tasks (e.g., 5-minute focus blocks, quick stretches, email triage).

Condition: 100% completion required.

Day 2: "The Royal Guards" (Intensity Test)

Quests: 3 High-Difficulty / Deep-Work tasks (generated by ai gemini).

Condition: 100% completion required.

Day 3: "The Blood-Red Commander" (The Boss Room)

Quests: 1 "Boss Level" task generated directly from the player's "Biggest Challenge" (defined in Onboarding), plus their standard 3 daily habits.

Condition: Must be completed before midnight. No Penalty Shields allowed.

Failure State: If the player fails any day, the quest ends. They are thrown into the Penalty Zone, and the Job Change Quest goes on a 7-Day Cooldown.

4. The Evaluation Logic (Class Assignment)
Context: If the player survives Day 3, the System analyzes their lifetime data to assign a Class. The player does not choose; the System decides based on their actions.

Requirements:
The ProgressionService executes a calculateJobClass() algorithm looking at the player's highest attributes and PlayerHistory dossier.

Archetype A: The Vanguard (Combat/Physical Focus)

Condition: (STR + VIT) > (INT + SEN) by 20% or more.

Class Assigned: Silver Knight or Berserker.

Buff: +15% XP multiplier on PHYSICAL quests.

Archetype B: The Scholar (Cognitive/Focus Focus)

Condition: (INT + SEN) > (STR + VIT) by 20% or more.

Class Assigned: Arcane Mage or Grand Architect.

Buff: +15% XP multiplier on COGNITIVE quests.

Archetype C: The Shadow (Balanced/High Performance)

Condition: Stats are balanced within 10% of each other, AND the player cleared the 3-Day Gauntlet with zero delayed tasks.

Class Assigned: Shadow Necromancer (Evolves to Monarch later).

Buff: +5% XP multiplier globally. Unlocks the "Shadow Extraction" feature.

5. Rewards & Database Updates
Context: Surviving the gauntlet fundamentally changes the app's economy and look.
- Player will be granted a new archtype(or job title)
- Player will be granted 20 stat points
- player will be granted 2 A rank items (whcih can be used by him, haven't decided on the items yet, suggest some)
- player will be granted customisable theme
- player will be grnated with 50,000 gold 
Requirements:

Database Updates (player_identity):

Add column job_class (String).

Add column class_multiplier (JSON/Float).

The "System Title": The player's default title changes from "Novice Hunter" to their new Class name.

Thematic UI Unlock: * Automatically gift the player a premium UI Theme matching their class (e.g., Shadow Necromancer unlocks the 50,000 Gold "Shadow Theme" for free).

The frontend reads the job_class from the /api/player/status-window payload and permanently alters the default color variables (e.g., Red for Vanguard, Blue for Scholar, Purple for Shadow).
