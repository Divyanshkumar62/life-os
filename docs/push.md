3. Backend Requirements (Spring Boot)
   FCM Token Management:

Add a fcm_token field to the PlayerIdentity table.

Implement an endpoint `PUT /api/player/update-fcm-token` (request body with `playerId` and `token`) to store the unique device token generated by Android. The backend already contains `FcmController` which handles this route and updates the `fcm_token` column; ensure your client calls it as soon as the user signs in.

Firebase Admin SDK Integration:

Configure the FirebaseApp using your service account JSON. The backend will look for a file named
`docs/serviceAccountKey.json` by default, but will also accept any file in `docs/` whose name
contains `firebase-adminsdk` (e.g. the one you just added). You may alternatively set the
`FIREBASE_SERVICE_ACCOUNT` environment variable to the absolute path of the JSON file.

Event-Driven Dispatcher:

Attach a "Push Listener" to the SystemEvent creation logic.

Payload Logic: Every push must include "Data Payload" (JSON) for deep-linking (e.g., {"screen": "dungeon", "id": "123"}).

4. Android Client Requirements (Android Studio)

Background Handling:

Implement a FirebaseMessagingService to handle incoming messages even when the app is closed.

5. What I will provide to you
   To make this live, you need to handle the "infrastructure" setup on the Google side:

Firebase Project: Go to the Firebase Console, create a project, and add an "Android App" to it.

google-services.json: Download this file from Firebase and place it in your Android Studio app/ folder.

Service Account Key: Go to Firebase Settings -> Service Accounts -> Generate New Private Key. You will need to provide this .json file to your Spring Boot backend so it has permission to send messages.

Device Token Implementation: Ensure your Android code calls `FirebaseMessaging.getInstance().getToken()` when the user logs in, then POST that value along with the player's UUID to the backend endpoint. Example Kotlin snippet:

```kotlin
FirebaseMessaging.getInstance().token
    .addOnCompleteListener { task ->
        if (!task.isSuccessful) {
            Log.w("FCM", "Fetching FCM registration token failed", task.exception)
            return@addOnCompleteListener
        }
        val token = task.result
        // send to server
        lifecycleScope.launch {
            val body = mapOf("playerId" to playerId.toString(), "token" to token)
            apiClient.put("/api/player/update-fcm-token", body)
        }
    }
```

Keep the token updated whenever it changes.
